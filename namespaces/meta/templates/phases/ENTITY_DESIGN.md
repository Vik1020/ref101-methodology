# Phase: ENTITY_DESIGN

> Определение центральной сущности, проходящей через процесс

## Контекст

**Entity** — это бизнес-объект, который проходит через все States процесса.

Примеры Entity:
- Document (документ, согласуемый в процессе)
- Order (заказ, обрабатываемый в системе)
- Ticket (тикет, проходящий через поддержку)
- Campaign (кампания, запускаемая маркетингом)
- Release (релиз, проходящий через разработку)

**Доступные типы:**
- `string` — текст
- `number` — число
- `boolean` — да/нет
- `date` — дата
- `enum [val1, val2, ...]` — перечисление
- `object` — вложенный объект
- `list[type]` — массив

## Пример заполнения

```yaml
entities:
  - id: "document"
    type: "Document"
    role: "primary"
    schema:
      id: "string (required)"
      title: "string (required)"
      content: "string (required)"
      author_id: "string (required)"
      status: "enum [draft, review, approved, rejected] (required)"
      created_at: "date (required)"
      updated_at: "date (required)"
      review_score: "number (nullable)"
      comments: "list[Comment] (nullable)"
```
